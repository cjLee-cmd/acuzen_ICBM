## 🚀 빠른 시작(3분 요약)

1) 브라우저에서 시스템 주소를 엽니다 → 2) 이메일/비밀번호로 로그인합니다 → 3) 대시보드에서 통계를 확인합니다 → 4) 좌측 메뉴 ‘부작용 사례’에서 새 사례를 등록하거나 기존 사례를 확인합니다 → 5) 검토자/관리자는 사례 상세에서 ‘AI 분석 실행’ 버튼으로 분석을 요청하고 결과를 검토합니다 → 6) 관리자라면 ‘사용자 관리’, ‘감사 로그’, ‘시스템 모니터링’ 메뉴를 통해 운영을 관리합니다.

필수 기억 포인트
- 역할별로 보이는 메뉴가 다릅니다(일반 사용자/검토자/관리자).
- 환자 식별 가능한 정보(PII)는 꼭 필요한 경우에만 입력하세요.
- AI 분석은 참고용입니다. 최종 판단은 의료진이 내립니다.

## 🎯 시스템 개요와 대상 사용자

약물감시(Pharmacovigilance) 시스템은 의약품 부작용(ADR) 사례를 안전하게 접수·관리하고, AI 분석을 통해 위험도를 평가하며, 규제 준수에 필요한 감사 로그를 보관하는 플랫폼입니다.

주요 기능
- 부작용 사례 등록/조회/처리(상태 변경, 소프트 삭제/보관)
- AI 기반 위험도 평가 및 권고사항 생성(GPT-5 사용)
- 대시보드 통계/최근 사례/긴급 사례 확인
- 사용자/권한 관리, 감사 로그, 시스템 상태 모니터링

누가 사용하나요?
- 일반 사용자(USER): 사례 등록자(현장 의료진, 약사 등)
- 검토자(REVIEWER): 접수된 사례를 검토·보완·승인하는 담당자
- 관리자(ADMIN): 전체 운영(사용자/권한/감사/시스템), 보관 사례 접근

2) 이메일과 비밀번호 입력 → 로그인
3) 로그인 완료 시 대시보드로 이동

- 관리자: admin@pharma.com
- 검토자: reviewer@pharma.com
- 사용자: user@pharma.com

로그인이 안 될 때
- 이메일/비밀번호 재확인, Caps Lock 해제
- 계정이 비활성화된 경우 관리자에게 활성화 요청
- 비밀번호 분실 시 관리자에게 초기화 요청(현재 자체 재설정 기능은 제공하지 않음)

로그아웃

## 📊 대시보드 사용법

대시보드는 업무 현황을 한눈에 보여줍니다.

통계 카드(예시)
- 총 사례 수: 현재 등록된 사례 개수
- 대기 중인 사례: ‘검토 필요’ 또는 ‘처리중’ 상태의 개수
- 긴급/중요 사례: 높은 위험도(High/필요 시 Critical)로 분류된 사례 개수
- AI 정확도: 최근 예측 신뢰도의 평균(%)

최근 사례
- 가장 최근 등록된 사례 5건의 핵심 정보(사례 번호, 약물명, 위험도, 상태, AI 신뢰도)를 보여줍니다.
- 각 항목 클릭 시 해당 사례 상세 화면으로 이동합니다.

팁
- 대기 중인 사례가 많다면 ‘부작용 사례’ 메뉴에서 상태를 ‘검토 필요’로 필터링하여 먼저 처리하세요.

## 📝 부작용 사례 관리: 등록·조회·수정·보관

메뉴 경로: 사이드바 → ‘부작용 사례’

새 사례 등록(사용자/검토자/관리자)
1) 화면 우상단 ‘새 사례 등록’ 버튼 클릭
2) 환자 정보 입력
  - 나이: 숫자(예: 65)
  - 성별: 남/여/기타 중 선택
  - 주의: 환자 개인 식별 정보는 입력하지 않습니다
3) 의약품 정보 입력
  - 약물명/용량: 예) “타이레놀 500mg”, “아스피린 100mg”
  - 병용약(있다면): 간단히 기재(쉼표로 구분 가능)
4) 부작용 상세 입력
  - 증상, 발생 시점(복용 후 경과 시간/일), 지속 기간, 관련 검사 결과 등
5) 기타 정보(있으면)
  - 반응 발생 일자, 과거력, 결과(Outcome)
6) 저장 → 등록 완료(사례 번호 자동 생성: 예 CSE-2025-XXXXXX)

사례 목록 조회/검색/필터
- 검색: 사례 번호/약물명 일부만 입력해도 실시간 필터링
- 상태 필터: 전체, 긴급, 검토 필요, 처리중, 완료, (관리자) 보관 포함
- 정렬: 최근 등록 순(기본), 필요 시 열 헤더를 이용해 정렬(기관 UI 정책에 따름)

사례 상세 화면에서 할 수 있는 일
- 기본 정보 확인: 환자(나이/성별), 약물/용량, 반응, 설명, 상태, 위험도
- 관련 AI 예측 확인: 분석이 실행된 경우 최신 예측과 신뢰도 확인
- 상태 변경(검토자/관리자): 검토 필요 ↔ 처리중 ↔ 완료, ‘긴급’ 지정은 기관 정책에 따름

사례 수정/처리(검토자/관리자)
1) 사례 상세 화면 → ‘편집/처리’ 버튼
2) 변경할 항목 수정(예: 상태, 위험도, 결과 등)
3) 저장 → 변경사항 반영, 감사 로그 기록

사례 보관(소프트 삭제, 관리자 전용)
1) 사례 상세 화면 → ‘보관(삭제)’ 버튼
2) ‘보관 사유’ 필수 입력(규정 준수 목적)
3) 확인 → 사례는 보관 상태로 전환되고 기본 목록에 표시되지 않습니다
  - 주의: 보관 해제/복구 기능은 현재 제공되지 않을 수 있습니다(기관 정책 확인)

권한에 따른 보임 범위
- 사용자(USER): 본인이 등록한 사례만 확인 가능
- 검토자/관리자: 기관 정책에 따라 전체/배정 사례 확인 가능
- 관리자: 보관 사례 포함 열람 가능

## 🚨 긴급 사례(크리티컬) 확인과 대응

메뉴 경로: 사이드바 → ‘긴급 사례’

무엇을 보여주나요?
- 최근 30일 내 ‘High/Critical’ 위험도이며 상태가 ‘긴급/검토 필요/처리중’인 사례를 우선 정렬해 제공합니다.
- 각 항목은 경과일수, 최신 AI 예측(있다면) 등을 함께 보여 상황 파악을 돕습니다.

어떻게 대응하나요?
1) 가장 시급한 사례부터 클릭하여 상세 화면 이동
2) 즉시 필요한 조치가 있는지 확인(환자 안전 최우선)
3) 필요 시 AI 분석 실행(검토자/관리자)
4) 상태를 ‘처리중’으로 변경하고 후속 조치를 기록

## 🤖 AI 분석: 실행·결과 해석·활용

누가 실행할 수 있나요?
- 검토자(REVIEWER), 관리자(ADMIN)

어떻게 실행하나요?
1) 사례 상세 화면 오른쪽 상단의 ‘AI 분석 실행’ 버튼 클릭
2) 확인 팝업에서 진행 선택
3) 약 45~60초 처리(네트워크/모델 상황에 따라 다를 수 있음)
4) 결과가 화면에 표시되고, 예측 기록이 생성됩니다

결과는 어떻게 보나요?
- 위험도(Severity): Low/Medium/High/Critical, 신뢰도(%)
- 근거/리스크 요소: 판단 이유와 관련 위험인자
- 권장사항: 즉시 조치, 후속 조치, 규제 보고 필요, 추가 정보 수집 목록
- 처리 시간/모델 버전 정보

활용 팁
- AI 결과는 참고 자료입니다. 임상적 판단과 병행해 사용하세요.
- 권장사항 중 ‘추가 데이터 필요’를 체크리스트로 삼아 후속 조사에 활용하세요.

(선택) 예측 검토 기록
- 검토자/관리자는 예측에 대해 ‘휴먼 리뷰 완료’와 메모를 남길 수 있습니다(기관 UI 설정에 따름).

## 👥 사용자 관리(관리자 전용)

메뉴 경로: 사이드바 → ‘사용자 관리’

새 사용자 생성
1) ‘새 사용자’ 클릭 → 이름, 이메일, 조직 입력
2) 역할 선택: ADMIN / REVIEWER / USER
3) 초기 비밀번호 설정(안전한 전달 절차 준수)
4) 저장 → 생성 완료

수정/비활성/삭제
- 사용자 카드에서 ‘편집’으로 정보 변경, 활성화 토글로 로그인 허용/차단
- ‘삭제’는 복구가 어려우므로 신중히 사용

비밀번호 초기화
- 사용자가 분실한 경우 관리자가 임시 비밀번호를 발급(내부 절차 준수)

## 🧾 감사 로그(관리자 전용)

메뉴 경로: 사이드바 → ‘감사 로그’

무엇이 기록되나요?

## �️ 시스템 모니터링(관리자 전용)

메뉴 경로: 사이드바 → ‘시스템 모니터링’

확인 항목(예시)
- 시스템 상태 지표, 평균 응답 시간, 오류 추이
- AI 분석 서비스 응답 현황, 평균 처리 시간
- DB 연결/응답 추이(기관 구성에 따라 표시 항목이 달라질 수 있음)

점검 루틴
- 일일: 상태 지표/오류 확인
- 주간: 성능 추이/저장공간 점검
- 월간: 사용량 증가 추이와 확장 필요성 검토

## 🧷 입력 모범 사례(PII 최소화 등)

반드시 지켜주세요
- 환자 개인 식별 정보(이름/주민번호/연락처)는 입력하지 않습니다(필수 항목 제외)
- 약물명/용량은 표준 표기 사용(예: 성분명 + 용량)
- 반응 기술 시 객관적 사실/시점/경과를 명확히
- 병용약/과거력 등은 약어 남용을 피하고 풀네임 표기를 권장

## 🛠️ 문제 해결(자주 묻는 질문 포함)

로그인 실패
- 이메일/비밀번호 확인, Caps Lock 해제, 쿠키 허용 여부 확인
- 계정 비활성화 시 관리자에게 활성화 요청
- 비밀번호 분실 시 관리자에게 초기화 요청

사례가 보이지 않음
- 검색/필터 조건 해제(상태 ‘전체’로), 페이지 새로고침
- 사용자 역할상 접근 권한이 없는 사례일 수 있음

AI 분석이 느리거나 실패
- 보통 45~60초 소요, 일시적 네트워크 문제일 수 있으니 재시도
- 연속 실패 시 관리자에게 문의(서비스 상태 확인)

권한 부족/접근 거부
- 현재 역할 권한 범위를 확인(검토자/관리자 전용 기능인지)
- 필요 시 관리자에게 역할 변경 요청

브라우저 권장 설정
- 최신 버전 유지, 캐시 삭제 후 재시도, 다른 브라우저로 교차 확인

보안 유의사항
- 공용 PC에서 사용 후 로그아웃 필수
- 화면 캡처/내보내기 시 개인정보 노출 여부 재점검

에러 메시지가 나옵니다
- 표시된 메시지 전문과 수행 작업을 기록해 관리자에게 전달하세요

## 📚 용어 설명(Glossary)

- ADR(Adverse Drug Reaction): 의약품 이상반응(부작용)
- Pharmacovigilance: 약물감시(부작용 사례의 수집·평가·예방)
- KAERS: 국내 의약품 부작용 보고 시스템
- MFDS: 식품의약품안전처
- Severity(위험도): Low/Medium/High/Critical
- Case Status(상태): 긴급/검토 필요/처리중/완료
- 소프트 삭제(보관): 데이터는 유지하되 기본 목록에서 제외하는 방식(감사/규정 준수 목적)
